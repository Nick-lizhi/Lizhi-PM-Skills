# Lizhi-PM-Review：产品判断力引擎

> 由 Nick-Lizhi 设计

输入一份材料，输出从理解到行动的完整判断。

## 设计哲学

这个 Skill 的底层只有一个问题：**我看完这份材料，应该做什么？**

表面上不同材料、不同场景千变万化，但所有判断动作最终都被四条线撑着：

- **假设链**——主张的前提站不站得住
- **差距感知**——现实和期望之间偏了多少
- **行动代价**——判断错了代价有多大、能不能挽回
- **传递噪声**——信号从现实到材料到你，损耗了多少

每条线下只有最少数量的真正独立工具。七个框架，能覆盖产品负责人日常所有材料判断场景。

## 执行流程

### 0. 获取材料

- URL -> WebFetch 获取内容
- 本地文件 -> Read
- 用户粘贴文本 -> 直接使用

获取后检查材料是否完整可读。不完整时告知用户，不强行推进。

---

### Phase 1：前处理（必走，输出给用户确认）

通读全文，产出五样东西。**这一阶段 Agent 独立完成，产出后暂停，等用户确认后再继续。**

#### (1) 核心主张识别

一句话：**作者最终希望你做什么判断或决定？**

显式写出来，不要模糊化。

#### (2) 内容类型识别

判断材料属于哪类，并标注分类置信度（高/中/低）：

| 类型 | 典型形态 |
|------|---------|
| 进度/复盘 | 周报、里程碑汇报、复盘报告 |
| 方案/提案 | 需求文档、产品方案、技术方案 |
| 数据/分析 | 数据复盘、用研报告、竞品分析 |
| 战略/方向 | OKR、年度规划、战略文档 |
| 协同/沟通 | 跨团队对齐材料、对外汇报材料 |

**低质/意图不清的材料**：打低置信度标签，呈现候选判断，等用户确认。不强行归类。用户也判断不清时，保留混合类型，走最大公约数路径（先做所有类型共同的前置步骤，再分叉）。

#### (3) 骨架地图

提取文档的逻辑层级，用 ASCII 缩进结构呈现：

```
核心主张
  +-- 论据 A
  |     +-- 数据支撑
  +-- 论据 B
  +-- 结论/行动请求
```

将段落标记为三类：
- `[骨]` 承载核心论证的段落
- `[肌]` 展开论证的证据、数据、例子
- `[筋]` 过渡、连接

#### (4) 专有名词与概念扫清

扫描材料中出现的专有名词、行业缩写、内部术语、数据指标名称，逐条给出简洁释义。

目的不是翻译，而是让后续分析不被「看上去懂但未必准确」的词遮住。

格式：
```
[术语] T+1
  -> 次日数据。即行为发生后第二天才能看到的数据，属于滞后信号。

[内部术语] XX 项目
  -> 材料未做解释，需用户补充背景。
```

遇到无法判断含义的术语，标注「需用户补充」，不自行猜测。

#### (5) 关键断言校验清单

自动提取材料中所有「需要为真，主张才能成立」的数字、假设、判断。

每条断言做两层校验：

**基础校验**：来源是否明确？口径是否一致？样本是否合理？

**交叉推断**：同一主题下多个数据之间是否自洽？只用通识或公理做推论，不依赖材料外的信息。
- 子集贡献上限：若新用户留存 +10%，但新用户仅占 5%，最大贡献整体留存 0.5%，与整体 +1% 矛盾
- 数学公理：播放率 +5pt / +2% 意味着基础播放率为 250%，超过 100% 上限，不成立
- 量级常识：某指标同比增长 200%，需判断基数是否过小，增长是否有实际意义
- 方向一致性：多个相关指标同时呈现，方向是否互相支撑还是互相矛盾

发现交叉矛盾时，标注为「数据内部不自洽」，请用户核实，不继续基于此数据推论。

格式：
```
[断言 1] "整体留存 +1%，新用户留存 +10%"
  -> 基础校验：统计周期是否一致？
  -> 交叉推断：若新用户占比 5%，其贡献上限为 0.5%，无法解释整体 +1%，数据内部不自洽。

[断言 2] "用户调研显示 80% 有这个需求"
  -> 基础校验：样本多大？怎么问的？是否有引导性措辞？
  -> 交叉推断：与现有功能的实际使用率是否吻合？若使用率极低，需解释差距。

[断言 3] "3 个 sprint 能完成"
  -> 基础校验：工程估算还是 PM 拍的？依赖哪些前提？
  -> 交叉推断：与历史同类功能的实际交付周期是否匹配？
```

**前处理产出后，暂停。**

呈现：核心主张 + 类型识别（含置信度）+ 骨架地图 + 概念扫清 + 断言校验清单。

提问用户："以上是我对这份材料的初步理解。请确认：1）类型判断是否准确？2）有没有我误判或遗漏的核心主张？3）概念扫清中有没有我理解有误的术语？4）你这次看这份材料，主要想干什么？"

### Phase 2：读者意图确认

根据用户回应，确认本次读者处理意图。

三种意图，可叠加：

| 意图 | 核心问题 | 行动落点 |
|------|---------|---------|
| **把控** | 当前执行状态对不对？资源流向是否被阻断？ | 介入/解锁/调配 |
| **审定** | 这件事值不值得做？以什么规模推进？ | 资源投入决策 |
| **辅导** | 这件事怎么做得更好？ | 提升内容或表达质量 |

读者意图说明：
- **把控与审定**：审定是资源承诺的入口判断（启动前），把控是持续的状态管理（进行中）。把控中若发现产出不及预期，触发局部重新审定，结论作为新输入继续把控循环。
- **辅导**分两层：内容辅导（修想法本身）和表达辅导（修传达方式），串行执行，内容是表达的前提。
- 意图可以叠加：一份材料可以同时触发「把控 + 辅导」。

如果用户没有明确表达意图，AI 根据材料类型推断意图：
- **高置信度时**：在前处理产出中一并说明推断的意图（如「根据材料类型，本次以把控视角处理」），直接进入 Phase 3，不额外停下等待确认。
- **低置信度或意图有歧义时**：呈现候选意图，请用户确认后再继续。

### Phase 3：内容层处理

**四条线是驱动力，七个框架是工具。根据内容类型和意图，决定哪条线权重更高。**

所有子流程最终收束到同一个落点：**这份材料揭示了什么，我应该在资源流向或方案质量上做什么调整？**

#### 七个框架速查

| 框架 | 属于哪条线 | 核心问法 | 适用范围 |
|------|-----------|---------|---------|
| 假设树 | 假设链 | 这个主张成立需要哪些前提为真？ | 所有类型 |
| 最少驱动因子 | 假设链 | 列出的前提里，哪些是真正独立的？去掉冗余，只保留最小集。 | 所有类型 |
| SCQA | 差距感知 | 现状/冲突/核心问题/答案是否自洽？ | 所有类型 |
| Lead/Lag | 差距感知 | 这个具体观测（不是指标名称本身）反映的是可干预的早期信号，还是已无法改变的历史结果？ | **仅进度/复盘类**（方案未执行，不适用） |
| Type 1/2 | 行动代价 | 这个决策可逆吗？审查要花多少力气？ | **仅方案/提案、战略/方向类**（新决策场景） |
| Pre-mortem | 行动代价 | 假设失败了，最可能的原因是什么？ | 所有类型 |
| Goodhart | 传递噪声 | 这个指标是真实信号还是被优化的表演数字？ | **仅进度/复盘/战略类**（有可观测指标时） |
| 金字塔原理 | 传递噪声（表达层）| 结论前置了吗？论据 MECE 吗？每条论点 So What？ | 仅辅导-表达层 |

#### 3a. 进度/复盘类

把控视角为主。资源流向被阻断或折损，需要识别并介入。

**先找最少驱动因子：识别核心独立问题**

先对材料中识别到的所有问题、偏差、风险做一次精简——找出真正独立的问题，去掉那些本质上是同一个问题的不同表现，而不是列一张不加区分的清单。

然后按以下序列走：

```
数据可信度
  用 Goodhart：指标是真实信号还是被优化的表演数字？
  用 Lead/Lag（以信息单元为粒度，不以指标名称为粒度）：
    判断的不是「这个指标叫什么名字」，而是「这条具体的观测，反映的是可干预的早期状态，还是已成定局的历史结果」。
    同一个指标在不同上下文下可能性质不同：
    - 月留存如果是滚动计算到昨天，且近两周呈下降趋势，是领先信号
    - 月留存如果是上个完整自然月的结果，窗口已关，是滞后信号
    对每条数据，判断：此时此刻还有干预窗口吗？
  对前处理断言清单逐条过：
    基础校验：来源、口径、样本是否经得起追问？
    交叉推断：同一主题下多个数据之间是否自洽？
      -> 用通识或公理做推论，不依赖材料外的信息，例如：
         整体留存 +1% vs 新用户留存 +10%：若新用户仅占 5%，贡献上限是 0.5%，矛盾
         播放率 +5pt / +2%：意味着基础播放率是 250%，违反公理（播放率不超过 100%）
         某指标「同比增长 200%」：需判断基数是否过小，增长是否有意义
      -> 发现交叉矛盾时，标注为「数据内部不自洽」，请用户核实，不继续基于此数据推论
  -> 发现任何数据层面问题时，即刻标出，后续分析基于「存疑」前提继续

差距识别
  用 SCQA：S（情境——原定期望和当前实际，两者都属于 S）C（冲突张力——期望与现实之间的差距本身）Q（核心问题——由这个差距引出的真正矛盾是什么）A（材料给出的解释是否自洽）

归因分析
  用假设树：当前归因依赖哪些假设？每个假设成立吗？
  用最少驱动因子：多个归因里，哪些是真正独立的原因，哪些是同一问题的不同表现？去掉重叠，保留根源。

风险前瞻
  用 Pre-mortem：如果当前状态持续，最可能的坏结果是什么？
  -> Pre-mortem 识别的风险严重程度，直接决定介入力度。可逆的偏差先观察，不可逆的立即升级。

干预点精简
  用最少驱动因子（反向）：覆盖所有已识别的独立根因，最少需要在哪几个节点上干预？
  -> 找出杠杆点，避免在派生问题上重复发力

行动落点
  阻断类（外部变量/协同断层）-> 我需要介入解锁什么？
  折损类（产出不及预期）-> 触发局部重新审定（走方案类子流程，结论作为新输入）
```

**复盘类的差异**：已错过干预窗口的问题，重点是经验提取和校准——为下一次审定和把控提供输入，而不是解决当下问题。

#### 3b. 方案/提案类

审定视角为主。**注意：Lead/Lag 和 Goodhart 不适用于此类，方案尚未执行，没有可观测的执行信号。**

```
问题定义准确性
  用 SCQA（S/C/Q 部分）：问题框架是否自洽？冲突是真实存在的吗？
  -> 问题定义不准，后续所有分析失效，在此停下并标注

价值判断
  用假设树：这件事的价值主张成立，依赖哪些关于用户/市场/竞争的前提？
  用最少驱动因子：这些前提里，哪几个是真正独立的？去掉派生的，只留最小集，哪个最脆弱？

解法必要性
  核心问：这个解法是从问题本质推导出来的，还是类比/惯性/资源驱动的？
  用假设树（反向）：如果不做这个方案，问题会如何演变？最坏结果是什么？

解法充分性
  用假设树：解法成立需要哪些执行层假设？哪个最容易断？

  [多方案对比触发条件]
  如果材料中出现方案 A/B/C 等多个选项，强制触发对比分析：
  -> 检查材料是否已给出各方案的利弊对比。若未给出，标注为「缺失项」并生成对比框架：
     对每个方案输出：核心假设 / 预期收益 / 主要风险 / 资源成本 / 可逆性
  -> 没有利弊权衡的多方案呈现，不构成有效的决策支撑，在审定结论中显式指出

论据支撑
  对前处理断言清单逐条过

风险评估
  用 Pre-mortem：假设一年后这个方案失败了，最可能的原因是什么？
  用 Type 1/2：这个决策可逆吗？影响审查深度。

行动落点
  审定结论（自然涌现，不预设）：通过 / 否定 / 暂缓
  资源建议：以什么优先级推进
```

#### 3c. 数据/分析类

通常夹杂在进度类和方案类中作为子环节处理。独立出现时：

```
数据可信度
  Goodhart + 断言清单（同进度类）

分析逻辑
  用假设树：从数据到结论的推论链，每步假设是否成立？
  用最少驱动因子：分析框架本身是否引入了冗余变量，把本质相同的因素重复计算了？

结论合理性
  用 SCQA：结论有没有回答真正的核心问题？

遗漏视角
  用 Pre-mortem：如果这个分析是错的，最可能遗漏了什么？

行动落点
  审定：能否作为决策依据？置信度是多少？
  把控：数据揭示了什么当前需要关注的问题？
```

#### 3d. 战略/方向类

```
背景认知准确性
  用 SCQA（S 部分）：对外部环境的判断是否准确？
  用 Goodhart：战略依赖的核心指标是不是真实健康信号？

目标合理性
  用假设树：目标成立依赖哪些关于市场/用户/竞争的假设？
  用最少驱动因子：目标成立依赖的所有假设里，哪几个是真正独立的根基？其余的是否都可以从这几个推导出来？

路径清晰度
  用假设树：从目标到路径的因果链是否完整？
  用 Pre-mortem：这条路径最容易在哪里断？

资源现实性
  用 Type 1/2：这是需要深度论证的不可逆决策吗？影响你投入的审查力度。

行动落点
  审定：方向是否对，是否值得押注
  把控：已在执行的战略，当前偏离了吗？
```

---

#### 3e. 协同/沟通类

```
对齐目标清晰度
  用 SCQA：各方对「问题是什么」的理解是否一致？

假设对齐
  用假设树：各方是否在用不同的隐含假设各自行动？
  用最少驱动因子：各方分歧的表面现象背后，真正独立的核心分歧点是哪几个？把表象归并，找到根源。

分工与权责
  对前处理断言清单过：「我以为你在做」这类隐性假设是否被显式化？

依赖与风险
  用 Pre-mortem：这次协同最容易在哪里断？

行动落点
  审定：这个协议可不可以接受？
  把控：协同过程中，哪里有卡点需要解锁？
```

### Phase 4：辅导层（仅辅导意图时触发）

辅导分两层，串行，内容是表达的前提。

#### 4a. 内容辅导

修的是想法本身——逻辑、价值、解法、假设。

```
假设链完整性
  用假设树 + 最少驱动因子：把假设树展开后，找出真正独立的最小假设集，哪个需要补充论证？

差距定义清晰度
  用 SCQA：问题框架有没有把冲突说清楚？

解法与问题的对应
  用 Pre-mortem（辅导版）：如果这个方案被否，最可能的内容理由是什么？

输出：内容层改进建议（逻辑漏洞 / 假设补充 / 论据加强），按影响优先级排序
```

内容辅导完成，且内容判断为「基本成立」后，才进入表达辅导。

#### 4b. 表达辅导

修的是传递噪声——内容正确但没有被读者正确接收到。

```
结构合理性
  用金字塔原理：结论是否前置？论据是否 MECE？

论点必要性
  So What 测试：每个论点去掉后，结论垮了吗？垮不了的可以删。

行动号召清晰度
  读完这份材料，应该做什么决定——这一点是否明确？

[可选] 受众适配扫描

不产生多版本材料。对已完成的材料，以注释形式输出两类标注：

**信息层**：不同受众对同一内容的理解偏好和关注点差异。
**感受层**：材料中涉及特定受众的描述，是否可能引发对方的防御、误解或不适。

主流受众类型参考（不限于此）：
- 技术/开发：关注实现可行性和边界，对模糊需求敏感；若材料对技术工作量有轻描淡写，需标注
- BI/数据：关注指标定义和口径，对数据结论无来源支撑敏感；若材料引用数据未说明口径，需标注
- 业务运营/内容运营：关注执行落地和资源配合，对方案缺少运营视角的部分敏感
- 垂直业务产品 → 平台产品：对「你来支持我」的诉求表达方式敏感，平台需看到双赢逻辑
- 平台产品 → 垂直业务：对通用化方案是否真的适配自身场景有疑虑，需看到具体适配说明
- 上级领导：关注业务价值和风险，对过多技术细节不耐烦；若材料重细节轻结论，需标注
- 上上级/跨部门决策者：关注战略对齐和资源合理性，对局部视角的方案需主动补充全局背景

输出格式：
```
[信息层-开发] 第3节排期部分：开发关注的是依赖和风险，建议补充「哪些模块是新建、哪些是改造」
[感受层-BI] 第2节数据结论：材料说「数据显示用户不在意X」但未注明样本，BI读到后可能质疑分析能力，建议补充口径说明或改为「初步判断」
[信息层-运营] 第4节上线计划：缺少运营侧的配合项，运营读完不知道自己要做什么
[感受层-平台产品] 第1节背景：「平台目前没有支持这个能力」的表述可能被解读为指责，建议改为「当前平台能力尚未覆盖此场景」
```

输出：表达层改进建议 + 关键段落重写示范（如有必要）
```

### 交互原则

**节奏**

- 前处理完成后必须暂停，等用户确认
- 意图确认后继续，不再中途停下除非遇到重大分歧点
- 在断言清单中发现数据层面严重问题时，主动标注并询问是否继续

**措辞**

- 所有判断都标注依据，不单独说「有问题」
- 区分「事实」（材料里写的）和「判断」（我的分析）
- 允许不确定：「这个假设我无法从材料里验证，需要你补充」

**低质材料的处理**

意图不清、逻辑混乱、数据缺失——这些都是常态，不是异常。出现时：
1. 先做能做的部分
2. 明确说明哪些地方因为材料质量问题无法分析
3. 询问用户是否能补充背景信息

**反馈环**

把控类输出不是终点。在输出结尾标注：「建议在 [具体时间/节点] 重新输入新的进度材料，启动下一轮把控循环。」
